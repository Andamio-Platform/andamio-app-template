# Backlog Item Template

Use this template when creating GitHub Issues in Phase 3: TRIAGE.

---

## Issue Title Format

```
[{type}] {brief description}
```

Examples:
- `[UX] Improve enrollment button visibility`
- `[Bug] Transaction fails silently on timeout`
- `[Feature] Add course search functionality`

---

---

## Priority Scoring

**Impact (1-5):**
- 5: Blocks core functionality
- 4: Major UX issue affecting most users
- 3: Moderate issue affecting some users
- 2: Minor polish item
- 1: Nice-to-have enhancement

**Effort (1-5):**
- 5: Major refactor, multiple days
- 4: Significant work, full day
- 3: Moderate work, half day
- 2: Small change, 1-2 hours
- 1: Quick fix, < 1 hour

**Score Formula:**
```
Score = Impact * (6 - Effort)
```

**Priority Assignment:**
- P0: Score >= 15 (ship immediately)
- P1: Score >= 8 (ship this sprint)
- P2: Score < 8 (backlog)

## CLI Commands

### Create Issue

```bash
gh issue create \
  --title "[UX] {title}" \
  --body "$(cat <<'EOF'
{body content}
EOF
)" \
  --label "iteration:{date},type:ux,priority:p1"
```

### Add to Project Board (if configured)

```bash
gh issue edit {number} --add-project "{project_name}"
```
